Project Overview

Idea & Solution Description

The Problem

Running hardware-in-the-loop (HiL) test suites is often a manual, fragmented process. Engineers execute tests from the command line, dig through raw log files for results, and have no centralized view of what passed, failed, or needs attention. This slows down iteration and makes it harder to communicate test health to the broader team.

The Solution

The HiL Automation Dashboard is a desktop application built in Python that brings test configuration, execution, live logging, and results reporting into one unified GUI. Rather than replacing Pytest, it wraps it — giving the user a clean interface to select test suites, run them, and watch results populate in real time without ever touching a terminal.

Core Features

  • Test suite selection panel with checkbox-based configuration
  • One-click test execution powered by Pytest under the hood
  • Live execution log with real-time output streaming
  • KPI summary cards showing pass, fail, and skip counts
  • Results export to PDF and CSV for documentation and reporting
  • Modular architecture separating GUI logic from test execution layer

Tech Stack

Python Tkinter Pytest subprocess CSV Export PDF Report GitHub

Design Approach

The application follows a separation of concerns pattern where the GUI layer (Tkinter) never directly runs tests. Instead, it delegates execution to a backend module that invokes Pytest via subprocess and pipes output back to the log panel. This keeps the codebase modular, testable, and extensible for future hardware communication layers.
